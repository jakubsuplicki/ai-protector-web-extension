# Step 28 — Config Generation (RBAC + Limits + Policy Pack)

**Prereqs:** Step 27 (Tools & Roles CRUD)
**Spec ref:** agents-v1.spec.md → Req 3
**Effort:** 1.5–2 days
**Output:** Deterministic generation of rbac.yaml, limits.yaml, policy.yaml from DB state

**Module:** `src/wizard/` — adds services/config_gen.py, routers/config.py

---

## Why this step matters

This is the first moment user sees tangible output — real config files
generated from what they described in steps 1–4. If generated YAML is
wrong, the entire wizard value collapses.

**Design principle:**
- Klasyfikacja ryzyka → **rekomendacja** (user może nadpisać)
- Policy pack → **preset do wyboru** (user wybiera z listy)
- RBAC / limits / policy YAML → **generowane deterministycznie** (z tych danych zawsze ten sam wynik)

---

## Sub-steps

### 28a — rbac.yaml generator

Generate `rbac.yaml` from `agent_roles` + `role_tool_permissions` + `agent_tools` tables.

```yaml
# Generated by AI Protector for: Customer Support Copilot
# Generated at: 2026-03-10T14:30:00Z
# Agent ID: abc-123

roles:
  customer:
    tools:
      searchKnowledgeBase: { scopes: [read], sensitivity: low }
      getOrderStatus: { scopes: [read], sensitivity: low }
  support:
    inherits: customer
    tools:
      getCustomerProfile: { scopes: [read], sensitivity: medium }
  admin:
    inherits: support
    tools:
      issueRefund: { scopes: [write], sensitivity: high, requires_confirmation: true }
      getInternalSecrets: { scopes: [read], sensitivity: critical, requires_confirmation: true }
```

**DoD:**
- [x] `generate_rbac_yaml(agent_id)` → returns YAML string
- [x] Output includes metadata header (agent name, ID, timestamp)
- [x] Roles sorted by inheritance depth (base first)
- [x] Inherits-from chains resolved correctly
- [x] Scopes, sensitivity, confirmation all populated from DB
- [x] Tests: generate from seeded demo agent → compare against existing `rbac_config.yaml`
- [x] Tests: empty tools → valid YAML with empty roles
- [x] Tests: circular inheritance → error (not infinite loop)

### 28b — limits.yaml generator

Generate `limits.yaml` with per-role budgets derived from policy pack defaults + agent overrides.

```yaml
# Generated by AI Protector for: Customer Support Copilot
# Policy pack: customer_support

roles:
  customer:
    max_tool_calls_per_session: 20
    max_tokens_in: 8000
    max_tokens_out: 4000
    max_cost_usd: 0.50
  support:
    max_tool_calls_per_session: 50
    max_tokens_in: 16000
    max_tokens_out: 8000
    max_cost_usd: 2.00
  admin:
    max_tool_calls_per_session: 100
    max_tokens_in: 32000
    max_tokens_out: 16000
    max_cost_usd: 10.00
```

Default budgets by role tier (from policy pack):

| Tier | Tools/session | Tokens in | Tokens out | Cost |
|------|--------------|-----------|------------|------|
| Low (customer) | 20 | 8K | 4K | $0.50 |
| Medium (support) | 50 | 16K | 8K | $2.00 |
| High (admin) | 100 | 32K | 16K | $10.00 |

**DoD:**
- [x] `generate_limits_yaml(agent_id)` → returns YAML string
- [x] Defaults from policy pack, overridable per agent
- [x] Per-tool rate limits from `agent_tools.rate_limit`
- [x] Tests: generate with defaults → all values match pack
- [x] Tests: override one limit → only that value changes

### 28c — Policy pack templates

5 built-in packs as Python dataclass / YAML templates:

| Pack | Injection | PII | Secrets | Toxicity | Limits |
|------|-----------|-----|---------|----------|--------|
| `customer_support` | strict (0.3) | mask | on | moderate (0.6) | medium |
| `internal_copilot` | moderate (0.5) | log | on | lax (0.8) | high |
| `finance` | strict (0.2) | block | on | strict (0.4) | low |
| `hr` | strict (0.3) | block | on | strict (0.3) | low |
| `research` | lax (0.7) | off | off | lax (0.9) | very high |

**DoD:**
- [x] 5 pack templates as Python dataclasses (not YAML files)
- [x] `get_policy_pack(name)` → returns pack config
- [x] `list_policy_packs()` → returns all 5 with descriptions
- [x] Each pack has: scanner toggles, thresholds, limit defaults, redaction mode
- [x] Tests: all 5 packs load without error, contain all required fields

### 28d — policy.yaml generator

Generate `policy.yaml` from selected policy pack + agent overrides.

```yaml
# Generated by AI Protector for: Customer Support Copilot
# Policy pack: customer_support

scanners:
  injection_detection: true
  injection_threshold: 0.3
  pii_redaction: true
  pii_mode: mask        # mask | block | log
  secrets_scanning: true
  toxicity_detection: true
  toxicity_threshold: 0.6
  nemo_guardrails: true

output_filtering:
  pii_redaction: true
  secrets_scanning: true
  injection_detection: true
  max_output_size: 4000

confirmation:
  required_for_sensitivity: [high, critical]
  timeout_seconds: 300
```

**DoD:**
- [x] `generate_policy_yaml(agent_id)` → returns YAML string
- [x] Values come from selected pack + any per-agent overrides
- [x] Tests: generate with `customer_support` pack → all thresholds match
- [x] Tests: override one value → only that changes

### 28e — Generation API endpoint

```
POST /agents/:id/generate-config
→ {
    rbac_yaml: "...",
    limits_yaml: "...",
    policy_yaml: "...",
    generated_at: "2026-03-10T14:30:00Z"
  }
```

**DoD:**
- [x] Endpoint generates all 3 files in one call

- [x] Stores last generated config on agent record (JSONB column)
- [x] `GET /agents/:id/config` returns last generated config (cached)
- [x] `GET /agents/:id/config/download` returns .zip with 3 YAML files
